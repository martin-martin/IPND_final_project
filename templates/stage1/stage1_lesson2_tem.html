The Structure of HTML documents

The DOM (Document Object Model) is a convention of how code is written,
that allows browsers to interact with HTML documents.

There are certain guidelines on how to make code better readable (e.g. indentations)

Especially interesting for me was to understand, that since what HTML does,
is to give meaning to pieces of code, it is logical to also structure it in such a way.
Therefore every piece of text (or image etc.) should be contained within a "box".
Useful "boxes" are <div>, different heading tags (such as <h1>) and <p>.

Chrome's "Inspect Element" Feature

This is a beautiful feature that allows me to check on the source code of websites and much more:

  • in one window it shows the structure of the HTML document
  • in another window it shows the CSS styles
  • hovering over a HTML line of code displays the corresponding element higlighted.
    all elements are actually a square, therefore the box around it.
    this is useful to gradually dig down to locate where is the code for a specific element!
  • clicking one of those lines allows me to see the associated CSS style in the other window.
    here i can do awesome stuff:
    style features can be switched off and on and also changed!
    and the immediate effects are visible on the website
  • with File/Save Page As.../Webpage, Complete the changed website can be downloaded
