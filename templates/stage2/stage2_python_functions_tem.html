What is a function?

Making a function (define)

Before we can do anything with a function, we have to make it. This is called "to define a function", or, short and practical:  def .

Indentations are important, and this is all syntax already. write def. write function name. open parenthesis. (optional: write argument. make comma, repeat until break), close parenthesis. make colonoscopy. write code. write return. Looks like this: 

def public_talk(arg1, arg2):
    strong_arg = arg1 + arg2
    return strong_arg

Using a function (call)

then type this:

public_talk('taxes', 'retirement')

easy.

Now explain me what is a function?

Make it!

A function is a set of commands (or a collection of lines of python code), that is summed up inside of a name. This name is the function name and it needs to be defined using the def statement. Also the lines of code need to be written first, before we can use them. We do this by indenting after the def line, and all code we write now within this indentation level, will be part of us defining our function.

Note that whatever are the inputs to a function, they never get changed! Even if it's assigned to a same variable name inside. A variable inputted to a function always stays within the function. Only a return statement can produce output from the function (or a print statement). All else is in vain. Look at this:

!

What's important to remember?

  • Functions don't share the variables with the main program. They are like a little program inside the main program
  • Functions need to be defined before they can be used. Yes, this means also locally before in the code!
  • Defining happens in the indented block
  • What comes into the functions brackets are its parameters. This is what the function can work with
  • Functions don't need to return anything, but for the output to be anything than a print statement, they have to
  • A function ends with a return statement. Anything written after it is less useful than something really, really useless.
  • They can make complicated and especially repetitive things much easier and faster to solve
  • In order to use the power of a function, we need not only define it, but also call it

Let's define and call a function! 

def fun_function(weather, sleeping_bag):
    if weather == "good":
        if sleeping_bag:
            print "fun!"
    else:
        return weather

Now what can we do with this? Not much maybe. But we can get three different results from running this function (not considering any of the millions of errors we could produce).

Given that we have the two variables weather and sleeping_bag, that could be either generated somewhere else in our python journey, or we'd simply write them on the spot - we'd definitely have to input them as parameters into the function call's parentheses. like this:

fun_function("good", True)

This we could run, and it'd produce: fun! yay! : )

Here's the two alternatives, depending on our input:

fun_function("bad", whatever)

fun_function("good", False)

What would these function calls effectively produce? Nothing. But there's a difference between the two nothings, and we could exemplify this with the following:

print fun_function("bad", whatever)

print fun_function("good", False)

Our first example would (now that we ask for it) communicate to us a sensible assessment of the situation, while the second one would remain silent. Of course this is due to the return statement, which is how functions effectively produce output. We could fix the silence of number two just by adding: 

def fun_function(weather, sleeping_bag):
    if weather == "good":
        if sleeping_bag:
            print "fun!"
        else:
            return "better wait a bit"
    else:
        return weather

We could, if we prefer silence concerning bad news, also do the following: 

def fun_function(weather, sleeping_bag):
    if weather == "good":
        if sleeping_bag:
            print "fun!"
            return
        else:
            return "better wait a bit"

Because if, in an if loop, we can terminate the function with a return statement either inside, or outside. can save the else.

So here are some examples of functions for lists

a_list.index()

here's one build-in method in python:

a_list.index(value)

gives us the index position of the first occurrence of the value in a_list. if the value is not part of the list, it returns an error.

<value> in <list>

another method is:

<value> in <list>

this is quite an uncommon style for a python function. what it does is returning a boolean value of True or False depending on whether the value is in the list or not. "not in" does the opposite.

a_list.append()

append is a method! methods distinguish from functions through having a first part (the one before the period):

a_list.append()

append mutates the list.

+

Now, I know this is not a function. It just somehow found its way into here... (I don't want to be rude and throw it out. It's also kinda cute)

when using the  +  operator on two lists, it creates a new, concatenated list object. it does not mutate the original lists.

len(a_list)

len(a_list)

Works for any object that consists of elements. The function counts how many elements are inside. (it also works for strings, the output being the number of characters in the string)

Extra: while loops and for loops

Something I used to forget: the break statement in while loops follows an IF condition! 

if something:
    break

Sometimes it is more natural to use a while loop than a for loop, because the while loop keeps track of the index that it's working on automatically - you need to have this aspect of it, otherwise it  would run forever ! So if you're trying to get a result that is about the index, a while loop might seem more logical. And it's possible. But for loops also work, one just needs to remember to insert a count that keeps track of the index position that the loop is in. --> index_while_for.png
