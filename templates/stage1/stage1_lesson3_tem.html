the HTML side

The DOM (Document Object Model) is a convention of how code is written, that allows browsers to interact with HTML documents. There are certain guidelines on how to make code better readable (e.g. indentations). You can use Chrome's Inspect Element to see the html code of this and any page.

Especially interesting for me was to understand, that since what HTML does, is to give meaning to pieces of code, it is logical to also structure it in such a way. Therefore every piece of text (or image etc.) should be contained within a  box .  Useful   boxes   are  <div>, different heading tags (such as <h1>) and <p>.

And then there are semantic elements, that take this meaning thing even further. These  are   boxes   as  well, but they also carry information about what's within. I read a bit about them here, and I tried to use some in this document, but I didn't completely get it yet.

CSS and inheritance

The code in the image further up is written in CSS. This is a programming language that is often used together with HTML.
CSS is pretty neat because with this it's possible to make a lot of structuring and heaps of prettyness. Also, it can be done very compactly, and can be made transportable and applied on many different sites.
Although there are other possibilities, it's best practice to create a separate CSS file to store that code. Often it is called style.css

CSS style has a specific order of importance, that are stronger in descending order:

  • browser default
  • stylesheet (css in a separate file)
  • <style>...</style> inside the html doc
  • inline style in an element (should be avoided!)

CSS means CascadingStyleSheets. The "cascading" describes inheritance.
This means that the styles of parent tags get inherited by child elements, although if there are different styles assigned they take effect in the above order.

CSS syntax

CSS commands are written in the following syntax:

selector {
    style_rule: attribute;
}

If the selector is a HTML tag then this applies to all same tags present in the document.

 box  sizing

One thing I learned before, is that it is good to semantically structure the page  in   boxes   .  What belongs together should also be grouped together. For this we can use <div> and <span>, which are both block elements. Block elements  create   boxes   .  Which is good. But block elements take by default the whole space of the page from left to right. So in order to display something next to something else, while still keeping it in a  box , we need CSS.

Just in the way that other style attributes get assigned to a certain selector, we add:

selector {
    display: flex;
}

which allows those block elements, that are  sub  -  boxes   of  the selector element, to appear next to each other.

Since different browsers have different standards, we have to write this at the beginning to make this possible:

* {
    -webkit-box-sizing: border-box;
    -moz-box-sizing: border-box;
    -ms-box-sizing: border-box;

    box-sizing: border-box;
}

For displaying the flex- box  feature in Safari, it's necessary to aditionally add: "display: -webkit-flex;" into the selector's curly braces.

More on sizing: W3Schools

Extra Notes

  1.  Box  sizes can be given in percentages too. This is very useful because it allows the website to keep its formatting on different screen sizes. Also changes in window size do not make such weird effects.
  2. There's also the possibility to define max_width and max_height. When the  box  is smaller than the defined max_width, then its size changes when the window size does, when it reaches the defined max, it doesn't change anymore.
